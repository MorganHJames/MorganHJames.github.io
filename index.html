Character Creator

This is a character creator I made in an attempt to replicate the Nintendo Wii Mii creator. This is a PC application and allows you to create your own custom character. You can choose the height, weight, voice pitch, animation style, skin color, hair, eyes, nose, hats, facial hair, accessories, top, gloves, back accessory, waist accessory, pants and shoes. Each option can be colored anyway you like. There is also an option to randomise everything! After creating the character they are saved and appear in the character lobby where you can view, edit or delete any of your previous characters. You can even perform a role call which makes all of the characters for organised lines. There is a download link for the exe or you can view the source code on my github.

Emotional Speed Dating

Emotional Speed Dating is the result of a research project studying human emotion detection and machine learning. I created the study to the left which allowed me to understand how emotion detection works and thus enabled myself to create a game using Unity and C# that utilises emotion as an input. Within the game you see yourself participating in a speed dating event. The goal of the game is to find true love by empathising with your dates (mimicking their emotion). The game utilises a convolutional neural network as the main tool for emotion detection. You can read in full about the methods used in the study or have it summed up within the poster below.

All the way USA!

All the way USA! Is an Augmented reality mobile application that I created out of the desire to learn more about the individual states of the USA. The app is made possible through the 3rd party package Vuforia which handles most of the Augmented reality aspects of the app. A 3D map of the USA is displayed upon a dollar bill when the camera has one in sight. The map can be scaled through a pinching gesture. The map has toggleable features including state names, state abbreviations and capital cities. When the user presses a state it becomes centered and the other states disappear to reveal more information about the state including the flag of the state, the seal of the state and an article about the history and statistics about the state. There are also three tests that you can use to see how well you know the name, flag and capital of each state. During a test all of the information is hidden apart from either the name, flag or capital of a state and you have to select the corresponding landmass on your first try to get a point. The high score of the user is saved in the application and displayed over the testing buttons. Since the app was made with the free version of Vurifora I can not distribute it on the play store or app store without getting a subscription so the download link is for the android APK.

Koopa's Revenge

Koopa's Revenge was created for the Game Boy. The goal of the project was to create a game scene where both sprites and particle effects would be present. The game was written in C/C++ using Visual Studio and the NO$GBA emulator/debugger. This project was challenging because of the small amount of documentation for GBA programming and lacklustre debugging information. Whilst this was a challenge to create I believe I learned a lot about the fundamentals of hardware programming and had a great time in doing so. Loot Swipe

Loot swipe is a small mobile game that I created in under 24 hours using C# and Unity. The objective within the game is to collect loot by solving a maze of floating crates. The maze is created with Prim's algorithm and you can scale the complexity of the maze up or down. The biggest challenge after the maze generation was figuring out how to keep the maze on screen no matter the complexity. The game is published on the google play store and all of the source code is in the GitHub repository.

Yokai - Spirits of Battle!

Yokai - Spirits of Battle is a local multiplayer 3D fighting game. The game flow goes as follows: select your character, select the map to fight on and then proceed to beat your friends up for superiority. The theme and art style is inspired from monsters in Japanese folklore(Yokai). I was the sole programmer and developer for the game. I created all the UI elements, sourced the sounds, coded every script and put everything together within Unity. However, the 3D models for the character and stages were made by other members of the team. I created the game with a design oriented approach meaning that if another character was to be added it would be as simple as dragging and dropping the correct elements into predefined slots. The game is currently only available for PC and requires a controller to play. The download link is for an exe of the game.

Open University VR

I developed a suite of Virtual Reality simulations for education and training for health care with Unity. The workflow consisted of interpreting nursing procedures into Virtual Reality scenarios that make use of voice commands, haptic feedback, head, and hand tracking. The role required me to create many robust scalable reusable classes, components, and systems. Working in a small team within an emerging field of study meant I had to conduct my research into how to intuitively and comfortably implement different required interactions. The simulations were deployed to the android‑based Oculus Quest all‑in‑one platform, currently for a limited alpha release. The suite of simulations is unfortunately not available to the public and so the link is to the Learning innovation team website at the Open University.

website

The list of procedures that I created simulations for goes as follows: Injection technique, Observations, Blood Glucose measurement, Catheterization, Blood Service procedure, Respiratory examination, Nasogastric tube insertion, Venepuncture and Wound care there is even an onboarding simulation to ease the user into the experience.
